# 인시던트 대응 SOP

## 심각도 분류

| 등급 | 정의 | 대응 시간 | 사례 |
| --- | --- | --- | --- |
| **P0** | 환자 안전 직결 / 전면 다운 | 즉시 (5분 내) | 임산부 금기 차단 실패, CRITICAL 상호작용 미탐지, API 전면 5xx, DB 손상 |
| **P1** | 핵심 기능 일부 장애 | 30분 내 | 결제 실패, 로그인 불가, 처방 추천 응답 없음 |
| **P2** | 일부 사용자 / 비핵심 기능 | 4시간 내 | 통계 페이지 오류, CSV 내보내기 실패 |
| **P3** | 사용 편의성 | 다음 영업일 | UI 깨짐, 텍스트 오타 |

## 대응 플로우 (P0/P1)

### 1. 인지 (0~5분)

- 자동 알림 트리거:
  - Sentry — 신규 에러 5분 윈도 50건 초과
  - Fly health check — `/health` 3회 연속 실패
  - UptimeRobot — `api.ongojisin.co.kr` 다운
- 사용자 신고 — support@ongojisin.ai / 카카오톡 채널

### 2. 선언 (5~10분)

대응 채널 분리:
- 운영 슬랙 #incident-active 채널 새로 만들기
- 인시던트 매니저(IM) 1인 지정 — 의사결정 권한 집중
- 타임라인 즉시 기록 시작 (몇 시 몇 분, 누가 무엇)

### 3. 격리 (10~30분)

피해 확산 차단이 원인 분석보다 우선:

- 임상 안전 로직 의심 → `FEATURE_FLAG_AI_RECOMMEND=off` (환경변수) 로 AI 추천 끄기
- 결제 의심 → `/billing` 경로 503 차단
- 전면 장애 → [rollback.md](rollback.md)

### 4. 복구 (30분~)

[rollback.md](rollback.md) 또는 [db-backup-restore.md](db-backup-restore.md) 절차 진행.

### 5. 사용자 공지

- P0 — 30분 내, 카카오톡 채널 + 이메일 일괄 발송
- P1 — 1시간 내, 상태 페이지 공지

템플릿:
> 안녕하세요, 온고지신입니다. {시각} 부터 {증상} 장애가 발생하여 {시각}에 복구하였습니다.
> 원인은 {간단 설명} 입니다. 잘못 표기된 처방·진료 기록은 {조치} 하였습니다.
> 불편을 끼쳐 죄송합니다. 문의: support@ongojisin.ai

### 6. 임상 안전 로직 회귀 시 추가 절차

임산부 금기 / CRITICAL 상호작용 / 환각 처방 노출 등이 의심되면:

1. 영향 받은 사용자(한의사) 전체 목록 추출 — 사고 발생 시각 이후 처방 추천 조회 로그
2. 잘못된 응답을 받은 케이스 ID 모두 추출
3. 해당 한의사들에게 **직접 전화** — "이 처방을 환자에게 전달하셨습니까?"
4. 환자에게 전달된 케이스가 있다면 — 임상 자문 한의사와 함께 안전 조치 검토
5. 식약처 보고 의무 검토 (의료기기 인증 진행 단계에 따라 다름)

## 사후 (24~72시간 내)

- Postmortem 작성 (`docs/ops/postmortems/`)
- 재발 방지책 → 코드/테스트/모니터링 어디에 가드를 박을지 명시
- 같은 종류 사고가 두 번째라면 — 임시 조치가 아니라 아키텍처 변경 검토
